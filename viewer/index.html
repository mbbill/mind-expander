Legend & shortcuts

Arrows

canonical structural reference between types — field, parameter, return, trait impl. The graph's default edge.
selected an edge you explicitly revealed (clicked a row's → glyph or a call-picker entry).
drift_below structural drift — the owned type lives deeper in the module tree than its owners would suggest. Common smell: an inner helper that wants to be promoted closer to its callers.
drift_above structural drift — the owned type sits in a module above where its owners are concentrated. Often a candidate for moving the definition closer to where it's actually used.

Call markers on a row

Function and method rows show small → glyphs on each side. Click either side to reveal / hide the matching arrows in the diagram.

→
left arrow — incoming someone in the workspace calls this function. Grey when no incoming arrows are currently revealed; turns blue and bold once you've revealed at least one.
→
right arrow — outgoing, local-only every call this function makes stays inside its own module. Grey = "nothing surprising over here".
→
right arrow — outgoing, external at least one call crosses a module or crate boundary. Blue mirrors the stroke color used for the actual cross-module arrows in the diagram.
→
right arrow — outgoing, unresolved at least one call couldn't be resolved to a known workspace symbol — typically a heuristic match on a shared method name, or a call into an external library.

Visibility dots

Color encodes how widely an item is callable. Same palette is used on the diagram's type dots and the minimap.

pub Rust pub declaration. TS exported top-level item or a public class member.
pub(crate) Rust visible within the same crate. TS n/a — TypeScript has no crate boundary.
pub(super) Rust visible to the parent module. TS protected class member (visible to subclasses).
pub(in path) Rust visible only within a named module path. TS n/a.
private / local Rust no pub modifier. TS non-exported top-level item, private class member, or #field.

Keyboard shortcuts

M
methodsshow or hide the per-type method buckets.
C
codeopen or close the right-side source-code panel.
S

R
resetreset everything — selections, expansion, focus, methods, zoom/pan.
Space
overviewtoggle fit-all overview vs. 100% at the cursor.
T
tour paneltoggle the floating tour-steps panel. Click any row in the panel to jump the bubble to that step. Toggling the panel off does not stop the tour.
